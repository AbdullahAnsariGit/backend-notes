üìö Web Architecture Overview

Web development ka structure teen main hisson mein takseem hota hai: Frontend, Backend, aur Database. Ye sab mil kar ek complete web application banate hain.

  • Frontend: Ye wo hissa hai jo user browser mein dekhta hai. Isme HTML (structure), CSS (styling), aur JavaScript (interactivity) shamil hote hain. Maslan, buttons, forms, aur animations frontend ka hissa hain.
  • Backend: Ye server-side logic ko handle karta hai, jo user ko directly nazar nahi aata. Backend languages jaise Node.js, PHP, ya Python istemal hoti hain. Ye data processing, API calls, aur business logic ke liye zimmaydar hai.
  • Database: Ye data ko store aur organize karta hai. Databases jaise MongoDB (NoSQL) ya MySQL (SQL) istemal hote hain. Ye user data, posts, ya koi bhi information save karta hai.
  • Server: Ye ek bridge ka kaam karta hai jo frontend aur backend ko connect karta hai. Ye user requests ko handle karta hai aur responses bhejta hai.
  • APIs: Application Programming Interfaces jo frontend aur backend ke darmiyan data exchange karte hain (e.g., REST, GraphQL).
Note: Frontend aur backend ke darmiyan communication aksar HTTP requests ke zariye hoti hai, aur APIs is communication ko structured banate hain.

üî∑ Node.js Introduction

Node.js ek open-source, cross-platform runtime environment hai jo JavaScript ko browser ke bahar chalane ke liye banaya gaya hai. Ye server-side applications ke liye istemal hota hai.

  • Definition: Node.js JavaScript ko server par chala sakta hai, jo pehle sirf browsers mein chalta tha.
  • Features: Ye fast, lightweight, aur real-time applications (chat apps, live updates) ke liye ideal hai.
  • Single-threaded: Node.js ek single-threaded, non-blocking I/O model par kaam karta hai, jo isay efficient banata hai.
  • Use Cases: Real-time apps (e.g., chat, gaming), APIs, microservices, streaming services, aur command-line tools.
Note: Node.js Google Chrome ke V8 JavaScript engine par chalta hai, jo isay bohat tezi deta hai.

üì¶ Node.js Modules

Node.js mein modules code ko organize aur reuse karne ka tareeqa dete hain. Har module ek file ya collection of files hota hai jo specific functionality provide karta hai.

  • Built-in Modules: Node.js ke sath ate hain, jaise fs, http, path, aur os.
  • Custom Modules: Tum apne modules bana sakte ho aur unhe require se import kar sakte ho.
  • Third-party Modules: NPM se install kiye jate hain, jaise express, lodash.

Creating a Custom Module:

// math.js function add(a, b) { return a + b; } function subtract(a, b) { return a - b; } module.exports = { add, subtract };

Using the Custom Module:

// app.js const math = require('./math'); console.log("Addition:", math.add(5, 3)); // Output: Addition: 8 console.log("Subtraction:", math.subtract(5, 3)); // Output: Subtraction: 2
Note: module.exports se functions ya objects export kiye jate hain, aur require se import kiye jate hain.

üì¶ Introduction to NPM

NPM (Node Package Manager) Node.js ka package manager hai jo libraries aur dependencies install karne ke liye istemal hota hai.

  • Initialize Project: npm init ek package.json file banata hai jo project ke metadata aur dependencies ko track karta hai.
  • Install Package: npm install se koi bhi package install kar sakte hain (e.g., npm install chalk).
  • Using a Package: Ek example chalk ke sath:
const chalk = require('chalk'); console.log(chalk.blue('Hello, this is colored text!'));
Note: package.json mein dependencies add karne ke liye npm install --save use karo.

üß† Node.js Architecture Explained

Node.js ki architecture isay unique aur powerful banati hai. Ye asynchronous aur non-blocking operations par mabni hai.

  • Single Thread: Node.js mein sirf ek main thread hota hai jo application ke primary tasks ko handle karta hai. Ye resources ko efficiently use karta hai.
  • Event Loop: Ye Node.js ka core hai jo asynchronous tasks ko manage karta hai. Jab koi task (jaise file read) complete hota hai, event loop callback ko execute karta hai.
  • Libuv: Ye ek C++ library hai jo I/O operations (file reading, network requests) ko handle karti hai. Ye cross-platform compatibility bhi provide karta hai.
  • Worker Threads: Intensive tasks ke liye Node.js worker threads provide karta hai jo multi-threading jaisa kaam karte hain.
Note: Event Loop ki wajah se Node.js ek hi waqt mein multiple tasks handle kar sakta hai bina block kiye.

üí° Non-blocking I/O in Action

Node.js ka non-blocking model isay allow karta hai ke wo ek task ke complete hone ka wait kiye bina dosre tasks ko handle kare.

const fs = require('fs'); fs.readFile('data.txt', 'utf8', (err, data) => { if (err) { console.error("Error reading file:", err); return; } console.log("File data:", data); }); console.log("Ye line pehle execute hogi kyunki readFile non-blocking hai...");

Is example mein, readFile asynchronous hai, isliye "Ye line pehle execute hogi" pehle print hoga, aur file ka data baad mein jab read complete hoga.

üåê HTTP Protocol Deep Dive

HTTP (HyperText Transfer Protocol) browser aur server ke darmiyan communication ka standard tareeqa hai.

  • Request Methods: Common methods hain GET (data retrieve), POST (data send), PUT (update), DELETE (remove).
  • Response Status Codes: Ye batate hain ke request ka kya hua (e.g., 200 OK, 404 Not Found, 500 Server Error).
  • Headers: Additional information provide karte hain, jaise Content-Type ya Authorization.
  • Stateless Nature: Har HTTP request independent hoti hai, yani server ko yaad nahi hota ke pehli request kya thi.
Note: Cookies ya sessions ke through state maintain kiya ja sakta hai.

üîÑ Synchronous vs Asynchronous in Node.js

Node.js mein do tarah ke operations hote hain: synchronous aur asynchronous.

  • Synchronous: Ye operations ek ke baad ek execute hote hain aur main thread ko block karte hain. Example:
    • const fs = require('fs'); const data = fs.readFileSync('hello.txt', 'utf8'); console.log("File content:", data); console.log("Ye line baad mein execute hogi...");
  • Asynchronous: Ye operations background mein chalte hain aur main thread ko block nahi karte. Example upar diya gaya hai.
Note: Asynchronous methods ko prefer karo kyunki ye performance ko behtar banate hain.

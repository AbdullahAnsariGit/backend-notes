E-commerce Project Notes

Hey kids! Let's learn about the cool parts of our e-commerce project (like an online shop) using Node.js! We'll talk about MVC Pattern, Authentication, and Role-Based Authorization in a fun and simple way!

1. MVC Pattern

What is MVC? MVC stands for Model-View-Controller. It's like organizing your toys into boxes to keep things neat and easy to find!

  • Model: The "data box" that stores information, like products and users in our online shop.
  • View: The "display box" that shows things to users, like a list of products on a webpage.
  • Controller: The "helper box" that decides what to do, like adding a new product when you click a button.

Why use MVC? It keeps our code organized, so we don’t mix everything up like a messy toy room!

In our e-commerce project: We use MVC to manage products and orders:

  • Model: Stores product details (name, price) and user details (username, password).
  • View: Sends JSON data (like product lists) to the user (we can add a webpage later).
  • Controller: Handles tasks like adding a product or creating an order.
Example: Folder structure for MVC
models/ → Stores product and user data
controllers/ → Code to add or update products
routes/ → Decides which controller to use

2. Authentication

What is Authentication? It’s like a secret password to enter a clubhouse! It checks if you are who you say you are.

How it works? In our shop, users log in with a username and password. We give them a special key (called a JWT token) to prove they’re allowed in.

  • Register: Create a new account with a username and password.
  • Login: Enter username and password to get a JWT token.
  • Use Token: Send the token with every request to prove you’re logged in.

In our e-commerce project: Only logged-in users can create orders, and we use a library called jsonwebtoken to make tokens.

Example: Login in our project
POST /api/login
{"username": "testuser", "password": "password123"}
Response: {"token": "xyz123", "role": "user"}

3. Role-Based Authorization

What is Authorization? It decides what you’re allowed to do. Like, only teachers can give homework, but students can’t!

Roles in our project: We have two roles: user and admin.

  • User: Can view products and create orders (like buying a toy).
  • Admin: Can add, update, or delete products and see all orders (like managing the shop).

How it works? After login, the JWT token tells us your role. We check if you’re an admin before letting you add products.

In our e-commerce project: We use middleware (like a gatekeeper) to block users from admin tasks.

Example: Admin-only task
POST /api/products (add a product)
Middleware checks: Is user an admin? If not, say "Access denied!"

Fun Fact: Our e-commerce project uses these ideas to make a safe and organized online shop, just like a real store where only managers can add new items!
